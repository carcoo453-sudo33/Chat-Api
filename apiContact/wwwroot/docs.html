Chat API Documentation

A realtime chat backend with REST endpoints and WebSocket (SignalR) events. All responses follow a standard envelope format. All entity IDs are GUIDs.

Base URL

http://localhost:5000

All API routes are prefixed with /api. WebSocket hub is at /hubs/chat.

Authentication

Most endpoints require a JWT Bearer token obtained from POST /api/auth/login.

How to authenticate:
Include the header Authorization: Bearer <accessToken> on every protected request.
Access tokens expire in 60 minutes. Use POST /api/auth/refresh to get a new one silently.
Demo credentials (in-memory seed):
alice / password123 (admin)  ·  bob / password123  ·  carla / password123
Rate limiting: Auth endpoints are limited to 10 requests/min per IP. File upload is limited to 20 requests/min. Global limit is 120 requests/min.

Response Format

Every REST response uses a standard envelope:

{ "success": true, "message": "Success", "data": { ... } or [ ... ], "total": 42 // only on list responses }

Error Handling

StatusMeaning
200OK — success
201Created — resource created
400Bad Request — validation failed
401Unauthorized — missing or invalid JWT
403Forbidden — insufficient role
404Not Found — resource missing
409Conflict — duplicate resource
429Too Many Requests — rate limit exceeded
500Server Error — unexpected failure (safe message only)

POST /api/auth/register

Create a new account. Returns JWT access + refresh tokens immediately.

{ "username": "john_doe", "email": "john@example.com", "password": "secretpass123", "displayName": "John Doe" }

POST /api/auth/login

Login with username or email and password. Returns access + refresh token pair.

{ "username": "alice", "password": "password123" }
Response 200
{ "success": true, "data": { "accessToken": "eyJ...", "refreshToken": "abc123", "expiresIn": 3600 } }

POST /api/auth/refresh

Exchange a valid refresh token for a new access token. Old refresh token is revoked (rotation).

{ "refreshToken": "abc123..." }

GET /api/auth/me

Returns the currently authenticated user's profile decoded from the JWT. requires auth

POST /api/auth/logout

Revokes the provided refresh token server-side and sets the user offline. requires auth

{ "refreshToken": "abc123..." }

POST /api/auth/change-password

Change password for the currently authenticated user. All existing sessions are revoked. requires auth

{ "currentPassword": "old", "newPassword": "new", "confirmPassword": "new" }

GET /api/users

Returns all registered users sorted by display name. requires auth

Response 200
{ "success": true, "data": [ { "id":"550e8400-e29b-41d4...", "username":"alice", "isOnline":true } ], "total": 3 }

GET /api/users/{id}

Fetch a single user by their GUID. requires auth

POST /api/users

Create a new user. Usernames must be unique. Returns the new user with a generated GUID. requires auth

{ "username": "john_doe", "displayName": "John Doe", "email": "john@example.com" }

PATCH /api/users/{id}

Partially update a user profile. requires auth

{ "displayName": "Johnny", "avatarUrl": "https://..." }

POST /api/users/{id}/status

Set online/offline/away status. Also broadcasts UserOnline / UserOffline via WebSocket. requires auth

{ "isOnline": true }

DELETE /api/users/{id}

Soft-deletes a user record (preserves history). admin only

GET /api/rooms

Returns all active (non-deleted) rooms sorted by latest message activity. requires auth

GET /api/rooms/user/{userId}

Returns only the rooms that include the given user as a member. requires auth

GET /api/rooms/{id}

Fetch a single room with member IDs and last-message preview. requires auth

POST /api/rooms

Create a new room. A URL-safe slug is auto-generated from the name via SlugHelper. type: 0 = Direct, 1 = Group, 2 = Channel. requires auth

{ "name": "engineering", "description":"Engineering team chat", "type": 1, "memberIds": ["550e8400-...", "6ba7b810-..."] }

POST /api/rooms/{id}/members

Add a user to an existing room. requires auth

{ "userId": "550e8400-..." }

DELETE /api/rooms/{id}/members/{userId}

Remove a specific user from a room. requires auth

GET /api/messages/room/{roomId}

Returns paginated message history, ordered oldest → newest. Soft-deleted messages show content as [Message deleted]. requires auth

Query ParamTypeDefaultDescription
limitint50Messages per page
skipint0Offset for pagination

POST /api/messages

Send a message. Automatically broadcasts ReceiveMessage to all room WebSocket subscribers. requires auth

{ "roomId": "550e8400-...", "senderId": "6ba7b810-...", "content": "Hello world!", "type": 0 // 0=Text, 1=Image, 2=File, 3=System }

PATCH /api/messages/{id}

Edit message content. Emits MessageEdited to room subscribers. Only the original sender can edit. requires auth

{ "content": "Updated message text" }

DELETE /api/messages/{id}

Soft-deletes a message. IsDeleted = true, content replaced with [Message deleted]. Emits MessageDeleted. requires auth

POST /api/messages/{id}/read

Mark a message as read by a specific user. requires auth

{ "userId": "550e8400-..." }

GET /api/messages/room/{roomId}/unread?userId={userId}

Returns the count of unread messages in a room for a user (excludes their own messages). requires auth

POST /api/files/upload

Upload a file using multipart/form-data. Max 20 MB. requires auth

Allowed types: .jpg, .jpeg, .png, .gif, .webp, .pdf, .txt, .zip, .mp4, .mp3
Form FieldTypeDescription
filefileThe file to upload required
roomIdstringIf provided, creates a message in this room
senderIdstringSender GUID for the auto-created message

DELETE /api/files/{fileName}

Delete a previously uploaded file by its stored filename (UUID + extension). Path-traversal safe. requires auth

GET /api/audit/recent

Returns the most recent audit events. admin only

Query ParamTypeDefaultDescription
countint50Number of events to return
Response 200
{ "success": true, "data": [ { "action":"auth.login", "username":"alice", "success":true, "timestamp":"2025-01-01T12:00:00Z" } ] }

GET /api/audit/search

Filter audit events by user, action key, and/or date range. admin only

Query ParamTypeDescription
userIdstring?Filter by user GUID
actionstring?Filter by action key (e.g. auth.login)
fromDateTime?Start of date range (UTC)
toDateTime?End of date range (UTC)
limitintMax results (default 100)

GET /api/audit/user/{userId}

Returns all audit events for a specific user. admin only

GET /api/audit/action/{action}

Returns all audit events matching an action key, e.g. auth.login, file.upload, file.delete. admin only

Audit action keys:
auth.register · auth.login · auth.login.failed · auth.logout · auth.refresh · auth.password.changed · file.upload · file.delete · file.delete.rejected

WebSocket — Connect

The WebSocket hub is powered by ASP.NET Core SignalR. Connections require a valid JWT — the hub has [Authorize] and reads identity from the token, not from client-supplied parameters.

Endpoint: /hubs/chat
Auth: Pass JWT as query string: ?access_token=YOUR_JWT
Presence: Connection/disconnection automatically broadcasts UserOnline / UserOffline to all clients. Identity is read from the JWT — no userId argument needed.

WebSocket Events Reference

Client → Server (invoke)

MethodArgumentsDescription
JoinRoomroomIdSubscribe to all messages in a room
LeaveRoomroomIdUnsubscribe from a room
TypingroomIdBroadcast typing indicator (identity from JWT)
StopTypingroomIdStop typing indicator
Ping—Test connection; server replies with Pong

Server → Client (on)

EventPayloadDescription
ReceiveMessage{ id, roomId, senderId, senderName, content, type, timestamp }New message in room
MessageEdited{ id, content }Message was edited
MessageDeleted{ id, roomId }Message was soft-deleted
UserTyping{ roomId, userId, displayName }User is typing
UserStoppedTyping{ roomId, userId }User stopped typing
UserOnline{ userId, connectedAt }User connected
UserOffline{ userId, disconnectedAt }User disconnected
UserJoinedRoom{ roomId, connectionId, joinedAt }Someone joined
UserLeftRoom{ roomId, connectionId, leftAt }Someone left
Pong{ timestamp }Reply to Ping

WebSocket JS Example

// Install: npm install @microsoft/signalr import * as signalR from "@microsoft/signalr"; const conn = new signalR.HubConnectionBuilder() .withUrl("/hubs/chat?access_token=YOUR_JWT") .withAutomaticReconnect() .configureLogging(signalR.LogLevel.Information) .build(); conn.on("ReceiveMessage", (msg) => { console.log(`[${msg.senderName}] ${msg.content}`); }); conn.on("UserTyping", ({ displayName }) => { console.log(`${displayName} is typing…`); }); await conn.start(); await conn.invoke("JoinRoom", "room-guid-here"); await conn.invoke("Typing", "room-guid-here"); // identity from JWT

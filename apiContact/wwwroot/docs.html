Chat API Documentation

REST + WebSocket (SignalR) backend. All responses use a standard envelope. All IDs are GUIDs. JWT required on protected routes.

Base: http://localhost:5000 Auth: Authorization: Bearer <token> WS Hub: /hubs/chat Rate limit: 10/min auth · 120/min global

Authentication

JWT access tokens · refresh token rotation · BCrypt hashing · rate limited to 10 req/min

Demo seed credentials: alice / password123 (admin)  ·  bob / password123  ·  carla / password123

Register a new user. Returns a JWT access token and refresh token immediately.

Request body

{ "username": "john_doe", "email": "john@example.com", "password": "secretpass123", "displayName": "John Doe" }

Validation

FieldRule
username3–32 chars, letters/digits/_ - .
emailvalid email, max 254 chars
password8–128 chars
displayNameoptional, max 64 chars

Login with username or email and password. Returns access + refresh token pair.

{ "usernameOrEmail": "alice", "password": "password123" }
Response 200
{ "success": true, "data": { "accessToken": "eyJ...", "refreshToken": "abc...", "expiresAt": "2025-01-01T13:00Z" } }

Exchange a valid refresh token for a new access token. Old refresh token is revoked (rotation). Each token can only be used once.

{ "refreshToken": "abc123..." }

Returns the authenticated user's profile decoded from the JWT claims.

Revokes the provided refresh token server-side and sets the user status to offline.

{ "refreshToken": "abc123..." }

Changes the authenticated user's password. All existing sessions are revoked after success.

{ "currentPassword": "old", "newPassword": "new_secure_pw" }

Users

Profile management, presence status, search, and soft-delete

Returns all non-deleted users ordered by display name. Full profiles (id, username, displayName, email, role, status, isOnline, lastSeen, createdAt).

Response 200
{ "success": true, "data": [ { "id":"uuid", "username":"alice", "isOnline":true } ], "total": 3 }

Full-text search across username, display name, and email with optional role and online-only filters.

Query params

ParamTypeDefaultDescription
qstring?—Full-text search
rolestring?—Filter by role (User/Admin)
onlineOnlyboolfalseOnly online users
pageint1Page number
pageSizeint20Max 100

Returns public profiles (no email) of all users with status Online.

Returns the full profile for a single user by their GUID. Returns 404 if not found or soft-deleted.

Partial update — only supplied fields are written. Owner or admin only.

{ "displayName": "Johnny", "avatarUrl": "https://..." }

Sets the full presence status. Also broadcasts UserOnline / UserOffline via WebSocket.

{ "status": 0 } // 0=Online 1=Away 2=Busy 3=Offline

Marks the user as deleted (IsDeleted = true). The record is preserved for audit trail and message authorship. The user can no longer log in.

Rooms

Group, Direct, and Channel rooms with auto-slug, tags, categories, and member management

Returns all non-deleted, non-archived rooms ordered by most recent message activity.

ParamTypeDescription
qstring?Search name, description, tags
categorystring?Exact category match (case-insensitive)
tagstring?Filter by a single tag
typeint?0=Direct 1=Group 2=Channel
includeArchivedboolInclude archived rooms
sortBystringactivity | name | createdAt | memberCount
directionstringasc | desc
page / pageSizeintPagination (pageSize max 100)

Returns a deduplicated, sorted list of all room categories in use.

Returns a deduplicated, sorted list of all tags across all rooms.

Returns all rooms where the authenticated user is a member, sorted by latest activity.

Returns the full room object including memberIds and last-message preview.

Lookup a room by its URL-safe slug (e.g. engineering-team). Case-insensitive.

Creates a new room. A URL-safe slug is auto-generated from the name (uniquified with a suffix if needed). Creator is automatically added as a member.

{ "name": "Engineering", "description": "Engineering team chat", "category": "tech", "tags": ["dev", "backend"], "type": 1, "isPrivate": false, "memberIds": ["uuid-1", "uuid-2"] }

Partially update name, description, category, tags, isPrivate, or isArchived. Changing the name re-generates the slug automatically.

Toggles the archived state. Use POST /api/rooms/{id}/unarchive to restore.

Adds the current user as a member. Private rooms require admin role.

Removes the current user from the room's member list.

Returns public profile objects (no email) for every member of the room.

{ "userId": "uuid" }

Removes the specified user from the room member list.

Soft-deletes the room. Messages are preserved for audit. The room stops appearing in listings and searches.

Messages

Send, edit, soft-delete, paginate, search, read receipts, and emoji reactions

Returns a page of messages ordered oldest → newest for display. Pagination walks newest-first then re-sorts. Soft-deleted messages show [Message deleted].

ParamTypeDefault
limitint50
skipint0
ParamTypeDescription
qstring?Full-text search in content
senderIdstring?Filter by sender GUID
tagstring?Filter by tag
typeint?0=Text 1=Image 2=File 3=System
from / toDateTime?Date range filter (UTC)
page / pageSizeintPagination

Returns the count of messages in the room that the current user has not yet read (excluding their own messages).

Persists the message and broadcasts ReceiveMessage via SignalR to all room subscribers.

{ "roomId": "room-uuid", "content": "Hello world!", "type": 0, // 0=Text 1=Image 2=File 3=System "tags": ["pinned"] }

Updates the message content and sets isEdited = true. Emits MessageEdited to room subscribers. Only the original sender can edit.

{ "content": "Updated message" }

Sets isDeleted = true, replaces content with [Message deleted]. Emits MessageDeleted. History is preserved for audit.

Adds the current user to the message's readBy set. Idempotent — safe to call multiple times.

{ "emoji": "👍" }

Removes the current user's reaction for the given emoji from the message.

Files

Local blob storage · UUID filenames · MIME validation · path-traversal safe · 20 req/min

Allowed types: .jpg .jpeg .png .gif .webp .pdf .txt .zip .mp4 .mp3  ·  Max size: 20 MB

Upload a file using multipart/form-data. Returns the stored filename (UUID + extension) and public URL. Rate limited to 20 req/min.

Form FieldTypeDescription
filefileThe file to upload required
roomIdstring?If provided, auto-creates a File message in this room
Response 201
{ "fileName": "550e8400-e29b.jpg", "url": "/uploads/550e8400-e29b.jpg", "size": 204800 }

Deletes a file by its stored name (UUID + extension). Path-traversal protected — only files within wwwroot/uploads/ can be deleted.

Static file served directly by ASP.NET Core. No auth required — use UUID filenames for obscurity. Files are stored under wwwroot/uploads/.

Tracked action keys: auth.register · auth.login · auth.login.failed · auth.logout · auth.refresh · auth.password.changed · file.upload · file.delete · file.delete.rejected
ParamTypeDefault
countint50
Response 200

ParamTypeDescription
userIdstring?Filter by user GUID
actionstring?e.g. auth.login
fromDateTime?Start of date range (UTC)
toDateTime?End of date range (UTC)

Returns all audit events associated with a specific user GUID, sorted newest first.

Filter by action key, e.g. auth.login, file.upload. Returns matching events sorted newest first.

WebSocket

ASP.NET Core SignalR hub at /hubs/chat · JWT-authenticated · automatic reconnect

Connect: ws://localhost:5000/hubs/chat?access_token=YOUR_JWT
Presence (UserOnline / UserOffline) is broadcast automatically on connect / disconnect — identity from JWT, no arguments needed.
MethodArgumentsDescription
JoinRoomroomIdSubscribe to a room's messages
LeaveRoomroomIdUnsubscribe from a room
TypingroomIdBroadcast typing indicator (identity from JWT)
StopTypingroomIdStop typing indicator
Ping—Test connection; server replies with Pong
EventPayloadDescription
ReceiveMessage{ id, roomId, senderId, senderName, content, type, tags, timestamp }New message
MessageEdited{ id, content }Message edited
MessageDeleted{ id, roomId }Message soft-deleted
UserTyping{ roomId, userId, displayName }User is typing
UserStoppedTyping{ roomId, userId }Stopped typing
UserOnline{ userId, connectedAt }User connected
UserOffline{ userId, disconnectedAt }User disconnected
UserJoinedRoom{ roomId, connectionId, joinedAt }Someone joined room
UserLeftRoom{ roomId, connectionId, leftAt }Someone left room
Pong{ timestamp }Reply to Ping
// npm install @microsoft/signalr import * as signalR from "@microsoft/signalr"; const conn = new signalR.HubConnectionBuilder() .withUrl("/hubs/chat?access_token=YOUR_JWT") .withAutomaticReconnect() .configureLogging(signalR.LogLevel.Warning) .build(); conn.on("ReceiveMessage", (msg) => { console.log(`[${msg.senderName}] ${msg.content}`); }); conn.on("UserTyping", ({ displayName }) => { console.log(`${displayName} is typing…`); }); conn.on("UserOnline", ({ userId }) => { console.log(`User ${userId} came online`); }); await conn.start(); await conn.invoke("JoinRoom", "room-guid-here"); await conn.invoke("Typing", "room-guid-here");
